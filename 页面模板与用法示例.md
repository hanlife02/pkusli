# pkusli 页面模板与用法示例

更新时间：2026-04-23

本文档整理当前项目内已经封装好的页面模板，重点说明：

- 最小导入方式
- 各页面适合什么内容
- 常用参数怎么传
- 可以直接复制到 `main.typ` 的示例

## 1. 最小使用骨架

如果你是从零开始写一份新演示，最小骨架可以直接参考下面这段：

```typst
#import "style/index.typ": presentation-theme
#import "slides/index.typ": (
  agenda-page,
  cover-page,
  end-page,
  references-manual-page,
  subject-content-page,
  subject-text-image-page,
  subject-triple-gallery-page,
  transition-page,
)

#show: presentation-theme

#let report-title = [我的汇报标题]
#let report-author = [你的名字]
#let report-institution = [你的单位]
#let report-date = [2026-04-23]
#let demo-image = "figures/background.png"

#cover-page(
  title: report-title,
  author: report-author,
  institution: report-institution,
  date: report-date,
)
```

建议把页面组织顺序保持为：

`封面 -> 目录 -> 分节过渡页 -> 正文页 -> 参考文献页 -> 结束页`

## 2. 已导出的页面模板

当前 `slides/index.typ` 暴露了这些常用页面入口：

- `cover-page`
- `agenda-page`
- `transition-page`
- `subject-content-page`
- `subject-text-image-page`
- `subject-triple-gallery-page`
- `references-page`
- `references-manual-page`
- `references-pages`
- `end-page`

如果只是写自己的汇报，优先从 `slides/index.typ` 导入，不要直接从 `style/` 层调用。

## 3. 模板说明与示例

### 3.1 封面页 `cover-page`

适用场景：

- 整套汇报首页
- 项目答辩封面
- 阶段总结汇报首页

常用参数：

- `title`: 主标题
- `subtitle`: 副标题，可省略
- `author`: 作者
- `institution`: 单位
- `date`: 日期

示例：

```typst
#cover-page(
  title: [多模态评测框架设计],
  subtitle: [阶段汇报],
  author: [Ethan],
  institution: [Peking University],
  date: [2026-04-23],
)
```

补充说明：

- `subtitle`、`author`、`institution`、`date` 传 `none` 时，对应区域不会显示。
- 这个模板也暴露了很多样式参数，比如 `title-size`、`line-width`、`meta-fill`，只有在你明确要改视觉细节时再动。

### 3.2 目录页 `agenda-page`

适用场景：

- 展示汇报结构
- 在正式内容开始前给听众建立预期

常用参数：

- `title`: 左侧卡片标题，默认是 `目录`
- `marker`: 左侧英文标记，默认是 `CONTENT`
- `sections`: 目录项列表，传入几项就按几部分排版，推荐 3 到 6 项
- `left-card-dx` / `left-card-dy`: 左侧卡片微调
- `right-card-dx` / `right-card-dy`: 右侧卡片微调

示例：

```typst
#agenda-page(
  left-card-dy: 1.5em,
  right-card-dy: 1.5em,
  sections: (
    [01 / 研究背景],
    [02 / 方法设计],
    [03 / 实验结果],
    [04 / 应用讨论],
    [05 / 总结与展望],
  ),
)
```

补充说明：

- `sections` 为空时，页面会显示占位提示。
- `sections` 为 3 到 6 项时，目录页会自动调整字号、行距和内边距。
- 如果目录项比较长，优先精简文案，不建议先缩小字号。

### 3.3 分节过渡页 `transition-page`

适用场景：

- 大章节切换
- 每一部分开始前给出主题和一句摘要

常用参数：

- `number`: 章节编号，如 `01`
- `marker`: 标签，默认是 `PART`
- `title`: 当前章节标题
- `description`: 当前章节一句话说明
- `left-card-dx` / `left-card-dy`
- `right-card-dx` / `right-card-dy`

示例：

```typst
#transition-page(
  number: [02],
  marker: [PART],
  left-card-dy: 1.5em,
  right-card-dy: 1.5em,
  title: [方法设计],
  description: [这一部分介绍系统结构、关键模块以及实现时的取舍。],
)
```

补充说明：

- 这是“章节页”，不是正文页，不适合塞太多信息。
- `description` 保持一到两句话效果最好。

### 3.4 通用正文页 `subject-content-page`

适用场景：

- 放结论、要点、步骤、短段落
- 作为最常用的正文页模板

常用参数：

- `title`: 页面标题
- `subtitle`: 可选副标题
- `top-content`: 顶部摘要条，一般写一句话结论
- `content`: 正文主体
- `header-left-offset`: 标题卡向右偏移量，常见值是 `10em`

示例：

```typst
#subject-content-page(
  title: [为什么先改 `main.typ`],
  header-left-offset: 10em,
  top-content: [一句话结论：多数情况下，你只需要先改入口文件里的变量和页面内容，不必一开始就改底层样式。],
  content: [
    推荐按下面顺序修改：

    - 先改封面信息
    - 再改目录结构
    - 然后替换正文内容
    - 最后补图片、参考文献和结束页
  ],
)
```

补充说明：

- `top-content` 对应页面上方的摘要卡，适合写一句话结论。
- `content` 里直接写 Typst 正文即可，列表、强调、短段落都能放。

### 3.5 图文页 `subject-text-image-page`

适用场景：

- 左文右图或左图右文
- 用截图、架构图、流程图配合文字解释

常用参数：

- 第一个位置参数：图片路径，例如 `"figures/background.png"`
- `title`
- `top-content`
- `text-body`: 文字区域
- `image-caption`: 图注
- `swap-sides`: 是否左右互换
- `text-width` / `image-width`: 两列宽度比例
- `header-left-offset`

示例 1：默认左文右图

```typst
#subject-text-image-page(
  "figures/background.png",
  title: [系统结构概览],
  header-left-offset: 10em,
  top-content: [一句话结论：把复杂结构先浓缩成一张图，再配合三到四个要点解释，页面会更稳。],
  image-caption: [图 1. 系统结构示意图。],
  text-body: [
    这一页适合承载：

    - 架构图
    - 流程图
    - 产品截图
    - 实验流程示意
  ],
)
```

示例 2：交换左右布局

```typst
#subject-text-image-page(
  "figures/background.png",
  title: [结果分析],
  header-left-offset: 10em,
  swap-sides: true,
  text-width: 1.1fr,
  image-width: 0.9fr,
  top-content: [一句话结论：当图片信息更重要时，可以把图片放到左侧。],
  image-caption: [图 2. 结果对比图。],
  text-body: [
    - 左侧展示核心图表
    - 右侧补充结论和解释
    - 适合“图先行，文后解释”的叙事方式
  ],
)
```

补充说明：

- 这个模板内部直接调用 `image(..image-args)`，因此图片路径建议按调用文件的位置来写；在 `main.typ` 里通常写 `"figures/xxx.png"` 即可。
- 如果你发现图片太小或太大，优先调整 `image-max-width` 和 `image-max-height`。

### 3.6 三图展示页 `subject-triple-gallery-page`

适用场景：

- 并列展示三个结果
- 展示三个模块、三个阶段、三张截图
- 用一页做“三项对比”

常用参数：

- `title`
- `image-one` / `image-two` / `image-three`
- `image-one-caption` / `image-two-caption` / `image-three-caption`
- `text-body`: 下方总结说明
- `header-left-offset`
- `image-height`
- `content-row-gap`

示例：

```typst
#subject-triple-gallery-page(
  title: [三个核心模块],
  header-left-offset: 10em,
  image-one: "figures/background.png",
  image-one-caption: [模块 A：数据处理],
  image-two: "figures/background.png",
  image-two-caption: [模块 B：模型推理],
  image-three: "figures/background.png",
  image-three-caption: [模块 C：结果展示],
  text-body: [
    这一页适合把多个对象并列摆出来，再在底部用一段话总结它们之间的关系、分工或者差异。
  ],
)
```

补充说明：

- 这个模板支持直接传字符串图片路径，也支持传自定义内容块。
- 内部做了路径归一化处理，所以像 `"figures/background.png"` 这种写法可以直接在 `main.typ` 里使用。
- 如果下方总结过长，先删冗余句子，再考虑减小 `body-text-size`。

### 3.7 参考文献页 `references-page`

适用场景：

- 你已经有 `references.bib`
- 需要直接根据 BibTeX 输出参考文献

常用参数：

- `sources`: BibTeX 文件路径
- `style`: 参考文献样式，默认是 `gb-7714-2015-numeric`
- `full`: 是否展开全部条目
- `title`
- `header-left-offset`

示例：

```typst
#references-page(
  sources: "references.bib",
  title: [参考文献],
  header-left-offset: 10em,
)
```

补充说明：

- 当你已经维护了 `.bib` 文件时，优先用这个模板。
- 如果条目较多，可能需要你自己控制参考文献数量，或者改用手工分页模板。

### 3.8 手工参考文献页 `references-manual-page`

适用场景：

- 参考文献不多
- 暂时不想维护 `.bib`
- 需要手工写说明性文献列表

常用参数：

- `entries`: 条目元组
- `start-number`: 起始编号
- `title`
- `header-left-offset`

示例：

```typst
#references-manual-page(
  entries: (
    [Touying package documentation. Typst Universe.],
    [Typst documentation. Typst 官方文档.],
    [项目仓库 README.md.],
  ),
)
```

补充说明：

- `entries` 为空时会显示占位提示。
- 这个模板很适合模板示例、课程汇报或阶段汇报。

### 3.9 多页手工参考文献 `references-pages`

适用场景：

- 手工文献条目较多
- 需要自动拆成多页显示

常用参数：

- `entries`: 全量条目
- `per-page`: 每页多少条
- `start-number`: 起始编号
- `title`
- `show-page-index`: 是否显示页码后缀

示例：

```typst
#references-pages(
  entries: (
    [文献 1],
    [文献 2],
    [文献 3],
    [文献 4],
    [文献 5],
    [文献 6],
    [文献 7],
  ),
  per-page: 4,
  show-page-index: true,
)
```

补充说明：

- 当条目超过一页时，这个模板比自己手拆页面更省事。
- `show-page-index: true` 后，标题会变成类似 `参考文献（1/2）`。

### 3.10 结束页 `end-page`

适用场景：

- 汇报收尾
- 答辩致谢页

常用参数：

- `title`
- `subtitle`
- `author`
- `institution`
- `date`

示例：

```typst
#end-page(
  title: [谢谢聆听],
  author: report-author,
  institution: report-institution,
  date: report-date,
)
```

补充说明：

- `end-page` 本质上复用了封面页的视觉结构。
- 如果你想保留统一风格，结束页优先只改文案，不改布局。

## 4. 选模板的简单规则

如果你不确定一页该用哪个模板，可以直接按下面判断：

- 纯文字结论、步骤、要点列表：用 `subject-content-page`
- 一张图配解释：用 `subject-text-image-page`
- 三张图或三组并列内容：用 `subject-triple-gallery-page`
- 大章节切换：用 `transition-page`
- 汇报结构总览：用 `agenda-page`
- 文献展示：优先 `references-page`，没有 `.bib` 时用 `references-manual-page`

## 5. 常见改动入口

多数情况下，真正需要频繁改动的只有这些位置：

- `main.typ` 顶部的标题、作者、单位、日期变量
- `agenda-page` 中的 `sections`
- 正文页中的 `title`、`top-content`、`content`、`text-body`
- 图文页和三图页里的图片路径与图注
- `references.bib` 或手工参考文献条目

如果你只是把模板换成自己的汇报内容，先改这些，通常已经足够。
