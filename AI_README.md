# AI Agent 使用说明

这个项目推荐使用仓库内 Skill：`skills/pkusli-slides`。

目标是让 AI Agent 用尽可能少的交互次数，把用户需求转成可编译的 Typst/Touying 演示稿。

## 首选方式

对 AI Agent 说：

```text
使用 pkusli-slides，把这个项目改成一份关于「你的主题」的演示稿。如果信息不够，请一次性问完需要的问题，然后更新 task.md、config.md 和 main.typ，并运行 typst compile main.typ 验证。
```

如果用户已经填写了 `task.md` 和 `config.md`，Agent 应直接读取并执行，不要重复询问信息。

## Agent 工作流

1. 先读取 `README.md`、`task.md`、`config.md` 和 `main.typ`。
2. 信息足够时直接修改，不要追问。
3. 信息不足时最多问一轮问题，并一次性问完。
4. 默认只修改 `main.typ`、`task.md`、`config.md`。
5. 默认不修改 `slides/`、`style/`、`figures/`。
6. 修改后运行 `typst compile main.typ`。
7. 最终回复说明修改内容、编译结果和输出文件。

## 一次性追问模板

```text
为了一次完成初稿，请补充下面信息。可以留空，留空的部分我会按正式中文汇报场景补全：

1. 演示主题是什么？
2. 目标受众是谁？
3. 使用场景是什么，例如课程汇报、项目汇报、方案宣讲？
4. 希望目录分为哪 3-6 部分？
5. 每部分有哪些必须出现的要点？
6. 是否有必须使用的图片、参考文献、作者单位或日期？
```

## 文件职责

- `task.md`：任务目标、背景、约束和验收标准。
- `config.md`：标题、作者、目录、正文要点、素材和偏好。
- `main.typ`：最终演示稿入口。
- `页面模板与用法示例.md`：页面组件参数和示例，仅在需要查页面用法时读取。

## 缺省规则

- 副标题缺失时使用 `none`。
- 图片缺失时复用 `figures/background.png` 作为占位图。
- 目录控制在 3 到 6 部分。
- 用户允许生成初稿时，可以自行补全合理内容。
- 除非用户明确要求，不调整主题样式和页面组件实现。
